🚀 Guia de Publicação

AtuAgendaPontual — tudo o que precisas fazer, por ordem, para lançar a app

1
Preparar os ficheiros Gratuito
Tens todos os ficheiros prontos — só precisas de os organizar
1
Cria uma pasta chamada atuagendapontual no teu computador
Dentro desta pasta, coloca os seguintes ficheiros que já tens:
• index.html → a landing page
• politica-privacidade.html → política de privacidade
• termos-uso.html → termos de uso
2
Cria o Gmail dedicado
Vai a gmail.com e cria uma conta nova: atuagendapontual@gmail.com (ou similar se já estiver tomado). Este email vai aparecer na Play Store.
2
Publicar o site no GitHub Pages Gratuito
O teu site fica em teuuser.github.io/atuagendapontual — necessário para a Política de Privacidade
1
Cria conta no GitHub
Vai a github.com → Sign up → usa o teu Gmail pessoal (não o novo). Username sugerido: atuagendapontual
2
Cria um repositório novo
No GitHub: New repository → nome: atuagendapontual → marca "Public" → clica "Create repository"
3
Faz upload dos ficheiros
Na página do repositório → "uploading an existing file" → arrasta os 3 ficheiros HTML → escreve "Primeiro upload" → clica "Commit changes"
Certifica-te que a landing page se chama exatamente index.html
4
Ativar GitHub Pages
No repositório → Settings → Pages (menu lateral) → Source: "Deploy from a branch" → Branch: main → pasta: / (root) → Save
Após 2-3 minutos o site fica disponível em https://atuatuagenda.github.io/atuagendapontual/
5
Guarda os URLs importantes
• Site: https://atuatuagenda.github.io/atuagendapontual/
• Política Privacidade: https://atuatuagenda.github.io/atuagendapontual/politica-privacidade.html
• Termos: https://atuatuagenda.github.io/atuagendapontual/termos-uso.html
O URL da Política de Privacidade é obrigatório para submeter a app na Google Play.
3
Configurar Firebase Gratuito
Autenticação e base de dados na cloud
1
Criar projeto Firebase
Vai a console.firebase.google.com → "Adicionar projeto" → nome: atuagendapontual → desativa Google Analytics → Criar
2
Ativar Authentication
Build → Authentication → "Começar" → Método de login → E-mail/palavra-passe → Ativar → Guardar
3
Criar Firestore Database
Build → Firestore Database → "Criar base de dados" → Modo de produção → Região: europe-west1 → Concluído
4
Configurar regras de segurança do Firestore
Firestore → separador "Regras" → substitui o conteúdo por:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid}/{doc=**} {
      allow read, write: if request.auth.uid == uid;
    }
  }
}

→ Clica "Publicar"
Sem estas regras os dados de todos os utilizadores ficam expostos!
5
Obter as credenciais da app
Definições do projeto (ícone ⚙️) → "Adicionar app" → Web (</>) → Nome: "AtuAgendaPontual" → Registar → Copia o objeto firebaseConfig e envia aqui na conversa
O firebaseConfig parece-se com isto:
{ apiKey: "AIza...", authDomain: "...", projectId: "..." }
4
Google Play Console 25€ único
Necessário para publicar a app
1
Criar conta na Google Play Console
Vai a play.google.com/console → "Começar" → paga os 25€ → preenche os dados de developer
Usa o teu Gmail pessoal (não o atuagendapontual@gmail.com) para a conta de developer
2
Criar a ficha da app
"Criar app" → nome: AtuAgendaPontual → português → App gratuita: Não → Aceitar políticas → Criar
3
Criar o produto in-app (pagamento)
Monetização → Produtos in-app → Criar produto:
• ID do produto: atuagendapontual_pro
• Nome: AtuAgendaPontual — Acesso Completo
• Descrição: Pagamento único com acesso vitalício a todas as funcionalidades
• Preço: 2,99€
• Estado: Ativo
O ID do produto atuagendapontual_pro tem de ser exatamente igual ao que está no código Android!
4
Preencher a ficha completa da app
• Título: AtuAgendaPontual — Gestão de Agendamentos
• Descrição curta: A agenda simples e completa para o teu negócio. Compromissos, clientes e faturação num só lugar.
• Descrição completa: (usa o texto da landing page)
• Screenshots: mínimo 2 capturas de ecrã da app (tira do teu telemóvel)
• Ícone: 512×512 px
• URL Política Privacidade: o URL do GitHub Pages que criaste
5
Gerar e publicar a APK Android Studio
Último passo antes do lançamento
1
Abrir o projeto no Android Studio
Descomprime o ZIP AtuAgendaPontual_Android.zip → Android Studio → "Open" → seleciona a pasta → aguarda Gradle sincronizar
2
Adicionar o ícone da app
No Android Studio: botão direito em res → New → Image Asset → escolhe o teu ícone → Finish
3
Gerar o AAB assinado
Build → Generate Signed Bundle/APK → Android App Bundle → Next → "Create new..." (cria a keystore) → preenche os campos → Next → Release → Finish
Guarda o ficheiro da keystore num lugar muito seguro! Se o perderes não podes atualizar a app.
4
Fazer upload na Play Console
Play Console → Produção → "Criar nova versão" → faz upload do ficheiro .aab → adiciona notas de versão → "Rever versão" → "Iniciar lançamento"
A primeira revisão da Google pode demorar 1 a 7 dias. As atualizações seguintes são mais rápidas.
